FileManager Class Documentation

Overview

The FileManager class is a utility for managing JSON files related to a project management system. It implements the FileHandler interface to provide methods for saving, reading, updating, and deleting data. This class leverages the Gson library for serialization and deserialization of data to and from JSON format.

Package

projectmanagement.utils

Dependencies

  • com.google.gson.Gson
  • java.io.*
  • java.util.ArrayList
  • java.util.Arrays

Fields

DATA_DIR

  • Description: Directory path for storing JSON files.
  • Value: "data/"
  • Access: Private, constant.

Methods

saveToFile

public void saveToFile(Object data, String filename)

Description: Serializes the given data object into JSON format and saves it to a file in the DATA_DIR directory.

  • Parameters:
    • data: The data object to be serialized and saved.
    • filename: The name of the file (without the .json extension).

readFromFile

public Object readFromFile(String filename, Class clazz)

Description: Reads and deserializes data from a JSON file into the specified class type.

Usage Examples

Saving Data to a File

FileManager fileManager = new FileManager();
User user = new User("John Doe", "john.doe@example.com");
fileManager.saveToFile(user, "user1");

Reading Data from a File

User user = (User) fileManager.readFromFile("user1", User.class);
System.out.println(user.getName());

Limitations

  • Files are stored as plain JSON without encryption.
  • Error messages are logged to the console but are not propagated to calling code.
  • Requires Gson library for JSON serialization and deserialization.

Recommended Improvements

  • Add support for custom directory paths.
  • Enhance error handling to throw exceptions or use a callback mechanism.
  • Include JSON schema validation to ensure data integrity.
